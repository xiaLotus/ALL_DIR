使用欄位完整說明

1. 資訊標題
範例：【F1 K22-8F 3390】
2. 投料場別 (Plant)
選擇 F1 / F3 / F5
3. 作業場別 (Proc Plant)
選擇 F1 / F3 / F5
4. 客戶碼 (需大寫)
範例：TZ
5. 站點(Operation)
範例：3400
注意事項：
  • 如多個，輸入完一站點後，按下Enter進入儲存空間，每輸入一個再按下Enter
  • 此欄位可搭配 sumK Function
  • 若需計算累計 K 數，請在「sumK 累計的站點」中填寫相應站點資訊
6. 作業機台編號(M/C ID)
範例：3390-A52
注意事項：
  • 如多個，輸入完mcid後，按下Enter進入儲存空間，每輸入一個再按下Enter
7. 使用的內部運行 Function
usegoal、NAgoal、nonegoal、sumK、delivery
每一個運行 Function 會根據資料進行不同型態的統計與報表輸出，請依據實際需求選擇對應代號進行運算設定。(暫定)
usegoal
範例：A51(Goal: 9) IN:18 OUT:20
如果要使用最基礎的個別站點及mcid，要輸出Goal，就直接使用這個 Function
NAgoal
範例：A52(Goal: NA) IN:18 OUT:20
如果要使用最基礎的個別站點及mcid，不輸出Goal，就直接使用這個 Function
sumK
範例：3190 1.406K
如果要計算K數，就使用這個 Function，如果要搭配站點加總，請往下選擇(比如：3191+3192)
delivery
範例：A05 IN: 11 OUT: 14 (派貨次數: 11)(每小時平均: 3.667)
這個選項不支援 Goal 設置
nonegoal
範例：5303 IN:7 OUT: 8
這個選項沒有多餘的資訊，只有 IN/OUT
8. sumK Funtion 累計的K數站點
站點相加，比如 3191+3192，輸入完後按下 Enter，後台會進行解析
9. 是否擬定 Goal
是否使用 Goal，如果需要就必須要填寫
10. Goal(可多個，但要按照排列；如果無，則默認 NA，選填)
如第9項為Y，則可以使用 各別的 Goal，請按照站點 or mcid 排序輸入相應的 Goal
11. 是否顯示總 Total
是否加總Total
12. 是否在總 Total 顯示 Goal
是否在總 Total 顯示 Goal
13. 總 Total 的 Goal
範例 Total (Goal:76.5) IN: 81 OUT: 84
這邊就是有設置 Total + Goal
14. 手動機台
按下Enter重複輸入手動機台(3670-3450-3800)
15. 自動機台
按下Enter重複輸入自動機台(3670-3450-3800)
16. 手動+自動機台的 Goal
手動+自動機台的 Goal
純自動機台 Total (Goal:70) IN: 59 OUT: 55
手動+自動機台 Total (Goal:70) IN: 66 OUT: 60
17. 幾小時發送一次
按照小時來當作分群基準
